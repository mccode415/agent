# Skill: Explain

## Trigger
`/explain [code/concept]` or "explain this" or "what does this do"

## Purpose
Explain code, concepts, or errors in clear, understandable terms.

---

## Process

### 1. Identify What to Explain

- Selected code block
- Error message
- Concept or term
- File or function

### 2. Analyze

For code:
```
## Code Analysis

### Overview
[1-2 sentence summary of what this code does]

### Step-by-Step
1. [First thing that happens]
2. [Second thing]
3. [Third thing]

### Key Concepts
- **[Term]:** [Explanation]

### Data Flow
[Input] → [Transform] → [Output]
```

For errors:
```
## Error Explanation

### What It Means
[Plain English explanation]

### Why It Happened
[Common causes]

### How to Fix
[Step-by-step fix]
```

### 3. Provide Context

```
### Related Concepts
- [Related thing 1]
- [Related thing 2]

### Common Patterns
[How this is typically used]

### Gotchas
- [Common mistake to avoid]
```

---

## Explanation Levels

Adjust based on context:

**Beginner:**
- Define all terms
- Use analogies
- Step-by-step detail

**Intermediate:**
- Assume basic knowledge
- Focus on specifics
- Mention edge cases

**Advanced:**
- Discuss internals
- Performance implications
- Alternative approaches

---

## Output Format

```
## Explanation: [topic]

### Summary
[1-2 sentence overview]

### Details
[Thorough explanation]

### Example
```[language]
[code example if helpful]
```

### See Also
- [Related documentation]
- [Related concept]
```
